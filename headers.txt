REQUEST HEADERS:
Cache-Control: no-cache
- Instructs intermediaries and the server not to use cached responses; forces a fresh request.

Postman-Token: <calculated when request is sent>
- A unique identifier generated by Postman for each request, used for debugging and tracking.

Host: <calculated when request is sent>
- Specifies the domain name of the server (www.googleapis.com) being requested.

User-Agent: PostmanRuntime/7.51.0
- Identifies the client software making the request (Postman version 7.51.0).

Accept: */*
- Tells the server the client will accept any content type in the response.

Accept-Encoding: gzip, deflate, br
- Specifies compression algorithms the client can decompress (gzip, deflate, Brotli).

Connection: keep-alive
- Requests the server to keep the TCP connection open for potential future requests.


RESPONSE HEADERS:
Content-Type: application/json; charset=UTF-8
- Indicates the response body is JSON data encoded in UTF-8.

Vary: Origin, X-Origin, Referer
- Tells caches that the response may differ based on these request headers.

Content-Encoding: gzip
- Indicates the response body is compressed using gzip.

Date: Tue, 20 Jan 2026 23:48:23 GMT
- The date and time when the response was generated by the server.

Server: ESF
- Identifies the server software (Google's ESF - Edge Service Frontend).

X-XSS-Protection: 0
- Disables the browser's built-in XSS filtering (modern approach uses CSP instead).

X-Frame-Options: SAMEORIGIN
- Prevents the page from being embedded in iframes on other domains (clickjacking protection).

X-Content-Type-Options: nosniff
- Prevents browsers from MIME-sniffing the response away from the declared Content-Type.

Alt-Svc: h3=":443"; ma=2592000,h3-29=":443"; ma=2592000
- Advertises that HTTP/3 is available on port 443 with a max-age of 30 days.

Transfer-Encoding: chunked
- Indicates the response is sent in chunks rather than with a predetermined Content-Length.
